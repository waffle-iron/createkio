All new Rails applications must use the same set of gems and tools, listed below, for development. Any applications that are bootstrapped by Createk.io, should make every effort to eventually use the same set of gems for development. Some development tools are required from day one! These are marked as bold.

When starting a new project, the latest stable version of the gem should be used. Gem version should be 'managed' when retro-fitting this to a running project. it is recommended to use a pragmatic approach to updating gems to the latest stable versions.

Gems

  • Puma - https://github.com/puma/puma
  • HAML - https://github.com/haml/haml
  • Airbrake
  • Rspec-Rails - https://github.com/rspec/rspec-rails
  • Capybara - https://github.com/jnicklas/capybara
  • Capybara-webkit - https://github.com/thoughtbot/capybara-webkit
  • Headless - https://github.com/leonid-shevtsov/headless
  • Database Cleaner - https://github.com/DatabaseCleaner/database_cleaner
  • Transactional Capybara - https://github.com/iangreenleaf/transactional_capybara
  • Shoulda Matchers - https://github.com/thoughtbot/shoulda-matchers
  • Factory_Girl - https://github.com/thoughtbot/factory_girl
